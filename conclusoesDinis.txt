Mapa 1a

O uso de uma função logaritmica para o sensor de obstáculos acabou por ser uma abordagem interessante para o mapa em questão, pois num mapa com muito espaço, o D31 andou muitas vezes com uma velocdidade alta. Velocidade essa que atingia maiores valores com uma função logaritmica para o sensor de recursos.

Mapa 1b

Neste mapa, com uma função logaritmica para o sensor de obstáculos, o D31 já teve um pouco mais de dificuldade em contornar os obstáculos, batendo por várias ocasiões nas paredes. A alta velocidade que no mapa anterior acabou por ser boa, neste mapa tornou-se um desafio. Ao combinar com uma função linear para o sensor de recursos, a travagem do D31 face aos obstaculos era dificultada pela alta velocidade com que chegava a um recurso, sendo por isso dificl de não bater nas paredes que tinham recursos muito próximos. Uma função gaussiana, melhorou um pouco, batendo apenas nos cantos das paredes. A função logaritmica para o sensor de obstaculos, apenas provou ser interessante quando combinada com a mesma função para o sensor de recursos. Ainda assim por vezes raspava nos cantos dos obstáculos.

Mapa 2a

Visto que o mapa 2a não apresenta obstáculos, optámos por desligar o sensor de obstáculos, usando apenas o sensor de recursos.
Usando uma função linear o D31 atinge o recurso sempre com o máximo de velocidade, o que proporciona que não consiga travar por completo antes de cair.
Uma função gaussiana foi particularmente interessante, porque foi possível de controlar o momento de acelaração e travagem do D31, através da média e variância.
Com uma função logaritmica foi possível também resolver o mapa, mas com um tempo inferior à gaussiana.
Nota: Visto que a função logaritmica para valores de x < 0.4 devolve um resultado maior que 1, o limite superior Y foi colocado em 2, para impedir que a função não se torna-se constante. O que resulta que o fator de peso aplicado ao valor do obstáculo fosse muito pequeno.

Mapa 2b

Novamente uma função logaritmica para o sensor de obstáculos foi muito desvantajosa num mapa que tem muitas paredes. Aliado à existência de buracos que permitiam o D31 cair ainda foi mais difícil encontrar uma configuração que funcionasse bem.
A melhor função para o sensor de recursos, aliada à logaritmica para obstáculos, acabou por ser uma linear, sendo que as outras duas acabaram sempre por bater. Ainda que a logaritmica foi só um toque na passagem estreita no início do mapa, pois no resto do mapa acabou por ter um desempenho muito mais aceitável do que as duas primeiras.
Nota: Na função linear foi usado um limite Y inferior de 0.2 para o D31 ter força suficiente para passar a primeira passagem estreita.

Mapa 3

Este mapa continha muitos obstáculos, pelo que o uso de uma função logaritmica para o sensor de obstáculos, acaba por ser difícil.



Notas Gerais:

Uma função logaritmica para o sensor de obstáculos, é proveitosa apenas quando temos um mapa sem muitos obstáculos e em que os recursos não se encontrem muito próximos das paredes. Para mapas com muitos obstáculos, torna-se difícil evitá-los.
Já para o sensor de recursos, uma função linear funciona bem quando temos recursos muito próximos uns dos outros e longe de paredes. Uma função logaritmica, é proveitosa na maior parte dos mapas, visto que se consegue atingir grandes velocidades quando o D31 está à procura de recursos.
